# BrewPlayer 部署指南：从零到上线

**目标：** 把 BrewPlayer 部署到 Cloudflare Pages，绑定域名 `player.礼字号.中国`

**前提：** 你已经有一个 Cloudflare 账号，并且 `礼字号.中国` 这个域名已经添加到 Cloudflare 里了。

**全程约需：** 15 分钟
**难度：** 零基础可操作，每一步都有截图级描述

---

## 第一步：确认你的 GitHub 仓库已就绪

BrewPlayer 的全部文件已经存放在你的 GitHub 仓库里，路径是 `packages/player/`。里面包含这些文件：

```
packages/player/
  ├── index.html          ← 主页面
  ├── player.css          ← 样式
  ├── player.js           ← 冲煮逻辑
  ├── manifest.json       ← PWA 配置
  ├── sw.js               ← 离线缓存
  ├── icon-192.png        ← 小图标
  ├── icon-512.png        ← 大图标
  ├── sample.brew         ← 示例方案
  └── seeds-manifest.json ← 50个种子索引
```

> 你不需要做任何操作，只需要知道这些文件在仓库里就行了。

---

## 第二步：打开 Cloudflare 后台

1. 用浏览器打开 `https://dash.cloudflare.com`
2. 输入你的邮箱和密码，点击「登录」按钮
3. 登录成功后，你会看到 Cloudflare 的首页（如果绑定了多个域名，会先看到域名列表）

---

## 第三步：进入 Pages 管理页面

1. 在 Cloudflare 后台的**左侧菜单栏**里，找到「**Workers 和 Pages**」这个选项
2. 点击它，会展开子菜单
3. 点击子菜单里的「**Pages**」

> 如果找不到，也可以直接在页面顶部的搜索框里输入"Pages"，然后点击搜索结果。

---

## 第四步：创建新的 Pages 项目

1. 点击页面上的「**创建**」按钮（蓝色按钮，通常在页面右上角）
2. 会跳出一个选择页面，点击「**Pages**」标签
3. 点击「**连接到 Git**」按钮

---

## 第五步：连接 GitHub

1. 系统会弹出授权页面，让你选择 Git 平台。点击「**GitHub**」。
2. 如果是第一次使用，Cloudflare 会要求你授权访问 GitHub。点击「**授权 Cloudflare Pages**」。
3. 在弹出的 GitHub 授权页面中，选择你的账号，然后点击「**安装**」或「**授权**」。
4. 授权完成后，页面会自动跳回 Cloudflare。

---

## 第六步：选择仓库

1. 在仓库列表中，找到 `aidulibrary/brewcode-os`
2. 点击这个仓库
3. 点击「**开始设置**」按钮

---

## 第七步：配置构建参数（重要！）

现在你会看到一个配置页面，有三个输入框需要填写。**请严格按以下内容填写**：

| 配置项（中文界面） | 填写内容           | 说明                  |
| :----------------- | :----------------- | :-------------------- |
| **项目名称**       | `brewcode-player`  | 随便起名，建议用这个  |
| **生产分支**       | `main`             | 默认就是 main，不用改 |
| **构建命令**       | 留空（什么都不填） | 纯静态网站不需要构建  |
| **输出目录**       | `packages/player`  | ⚠️ 这是最关键的一项！ |

> ⚠️ **特别注意「输出目录」这一项：** 必须填 `packages/player`，不能填项目名称，不能填别的。这是之前部署失败的原因。

填好之后，点击「**保存并部署**」按钮。

---

## 第八步：等待部署完成

1. 系统会自动开始部署，你会看到一个进度条
2. 等待大约 30 秒到 1 分钟
3. 部署完成后，页面会显示「**成功**」或绿色对勾
4. 你会看到一个临时域名，格式类似 `brewcode-player.pages.dev`

---

## 第九步：测试临时域名

1. 点击那个 `brewcode-player.pages.dev` 链接（或者复制到浏览器地址栏打开）
2. 你应该能看到 BrewPlayer 的页面，中间有「载入 .brew 文件，开始冲煮」的文字
3. 点击「**粘贴 JSON**」按钮，会展开一个文本框
4. 这说明部署成功！

---

## 第十步：绑定自定义域名 `player.礼字号.中国`

1. 回到 Cloudflare Pages 的项目页面（如果不在，左侧菜单 → Workers 和 Pages → Pages → 点击 `brewcode-player`）
2. 点击页面上方的「**自定义域**」标签
3. 点击「**设置自定义域**」按钮
4. 在输入框中输入：`player.礼字号.中国`
5. 点击「**继续**」或「**激活域**」按钮
6. Cloudflare 会自动完成 DNS 配置和 SSL 证书签发
7. 等待 1-2 分钟，状态会变成「**有效**」或绿色对勾

> 如果输入中文域名时遇到问题，可以用这个英文编码代替：`player.xn--fiq66k.xn--fiqs8s`

---

## 第十一步：最终验证

1. 打开浏览器，地址栏输入：`https://player.礼字号.中国`
2. 确认页面能正常打开
3. 确认地址栏前面有小锁图标（表示 SSL 证书生效）
4. 点击「粘贴 JSON」→ 粘贴任意一段 `.brew` 文件内容 → 点击「载入」
5. 确认能看到冲煮步骤引导

---

## 常见问题

### 问：部署失败了，提示「输出目录未找到」

**答：** 这是「输出目录」填错了。回到项目设置 → 构建配置 → 输出目录，确保填的是 `packages/player`（不是 `brewcode-player`，不是 `/packages/player`）。改完后点击「保存」，然后重新部署。

### 问：打开 `player.礼字号.中国` 显示空白页

**答：** 等 1-2 分钟，DNS 生效需要时间。如果超过 5 分钟还不行，检查 Cloudflare 后台的 DNS 记录里是否自动添加了 `player` 的记录。

### 问：页面打开了，但粘贴方案后点击「载入」没反应

**答：** 这是正常的。你需要粘贴一个完整的 `.brew` 文件内容。可以打开 GitHub 仓库里 `seeds/genesis-brews/` 文件夹，复制任意一个 `.brew.json` 文件的全部内容来测试。

### 问：后续更新了代码，怎么让网站也跟着更新？

**答：** Cloudflare Pages 会自动监听 GitHub 仓库的 `main` 分支。每次你 `git push` 新代码到 GitHub，Pages 会自动重新部署，无需手动操作。
