## 四、功能说明

### 4.1 AI 诊断

**交互流程：**

1. 点击工具栏「🔍 AI 诊断」按钮
2. 在弹出对话框中输入问题描述（如"这杯偏苦"）
3. 点击「提交诊断」，系统自动提取当前编辑器状态（brew JSON）作为上下文
4. AI 返回参数调整建议列表，每条包含：
   - `field`：字段名（中文）
   - `current`：当前值
   - `suggested`：建议值
   - `reason`：调整理由
5. 点击「应用建议」将建议值回填到编辑器

**字段匹配规则：** `applySuggestionToState()` 通过中文正则匹配字段名，支持：研磨度、水温、粉量、水量、粉水比、闷蒸时间、冲煮时间、器具、磨豆机。

### 4.2 AI 生成

**交互流程：**

1. 点击工具栏「🤖 AI 生成」按钮
2. 填写必填项：产地、烘焙度（下拉选择）、处理法（下拉选择）
3. 选填：豆种、设备列表（逗号分隔）、口味偏好
4. 点击「生成方案」
5. AI 返回完整 .brew JSON，自动替换编辑器所有字段
6. 如果在代码模式，自动切回表单模式

**烘焙度选项：** 浅烘、中浅烘、中烘、中深烘、深烘、极深烘
**处理法选项：** 水洗、日晒、蜜处理、厌氧发酵、湿刨法

## 五、技术细节

### 5.1 前端 AI 服务层

```javascript
// 配置常量
AI_API_BASE = 'https://api.礼字号.中国';
AI_API_KEY  = 'bk_test1234567890abcdef';
AI_TIMEOUT_MS = 15000;  // 15 秒超时

// 统一请求封装
aiFetch(path, body) → Promise<data>
```

- 使用 `AbortController` 实现超时控制
- 自动解析 API 错误信息（含 `detail` 字段）
- 超时、网络错误均有中文友好提示

### 5.2 加载状态

- 提交后按钮显示「诊断中…」「生成中…」并禁用
- CSS 旋转动画 + 半透明背景提示
- 请求完成后自动恢复按钮状态

### 5.3 安全措施

| 层级   | 措施                             |
| ------ | -------------------------------- |
| 网关   | D1 数据库鉴权，key_hash 匹配     |
| 网关   | KV 频率限制（10次/分钟）         |
| 前端   | 15 秒超时，防止无限等待          |
| Worker | 重试上限（诊断 2 次，生成 3 次） |
| Worker | 生成结果结构校验，不合格自动重试 |

### 5.4 成本

| 操作 | 输入 Token | 输出 Token | 成本   |
| ---- | ---------- | ---------- | ------ |
| 诊断 | ~1,000     | ~500       | ¥0.002 |
| 生成 | ~1,750     | ~1,500     | ¥0.005 |

> 模型：deepseek-chat，定价：输入 ¥1/百万 token，输出 ¥2/百万 token

## 六、部署依赖

### 6.1 D1 数据库

表 `api_keys` 需包含测试密钥记录：

```sql
CREATE TABLE IF NOT EXISTS api_keys (
  key_hash TEXT PRIMARY KEY,
  plan TEXT NOT NULL DEFAULT 'free',
  rate_limit INTEGER NOT NULL DEFAULT 10,
  created_at TEXT DEFAULT (datetime('now')),
  last_used_at TEXT
);

INSERT INTO api_keys (key_hash, plan, rate_limit)
VALUES ('-3e3230b0', 'test', 10);
```

> `-3e3230b0` 是 `bk_test1234567890abcdef` 去除 `bk_` 前缀后的哈希值。

### 6.2 Worker 环境变量

**diagnose / generate Worker：**

```toml
[vars]
DEEPSEEK_API_KEY = "sk-xxx"  # 有效的 DeepSeek API Key
```

### 6.3 部署命令

```powershell
cd D:\brewcode-os\workers\gateway   && npx wrangler deploy
cd D:\brewcode-os\workers\diagnose && npx wrangler deploy
cd D:\brewcode-os\workers\generate && npx wrangler deploy
```
